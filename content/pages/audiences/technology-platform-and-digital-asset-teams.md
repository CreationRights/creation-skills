# **Section 4: Company-Owned Production Agents**

## **Eyebrow**

PRODUCTION AGENTS

## **Headline**

Agents need system rules before they access company media.

## **Body**

Production agents can support media production, but platform teams need to know how those agents should access systems, files, metadata, and records.

A production agent should not have open-ended access by default.

It needs a defined role, owner, approved tasks, blocked actions, operating files, access limits, review path, and risk level.

## **Example Agents**

### **Asset Package Agent**

Checks whether final packages include required files, formats, captions, metadata, approval notes, and delivery materials.

### **Metadata Prep Agent**

Prepares draft metadata, naming suggestions, source references, usage notes, and missing-field flags for human review.

### **Reuse Review Prep Agent**

Flags original approval, usage scope, territory, channel, talent status, NILP® status, and rights questions before reuse.

### **Delivery Checklist Agent**

Checks delivery readiness, required formats, captions, subtitles, naming rules, approval status, and record completeness.

### **Archive Summary Agent**

Summarizes archive materials and flags missing source, rights, metadata, or reuse information before material moves forward.

### **Internal Handoff Agent**

Prepares structured handoff notes between creative, production, legal, brand, platform, DAM, MAM, and delivery owners.

# **Section 6: Access and Permissions**

## **Eyebrow**

ACCESS AND PERMISSIONS

## **Headline**

Agent access should be designed, not assumed.

## **Body**

Production agents may need access to creative briefs, asset packages, reference files, approved examples, metadata fields, transcripts, captions, and delivery requirements.

But access should match the agent’s job.

Creation Skills helps platform teams define what each production agent can see, prepare, suggest, export, or route.

## **Access Questions**

What files can the agent access?

What systems can the agent touch?

Can the agent read metadata?

Can the agent suggest metadata?

Can the agent create new assets?

Can the agent export files?

Can the agent access talent materials?

Can the agent access rights notes?

Can the agent access unreleased assets?

Can the agent prepare public-facing outputs?

Who reviews the output before it moves?

What record should be kept?

## **Outcome**

Agents can support production without creating uncontrolled access to media, records, or sensitive materials.

ASSET LIFECYCLE

## **Headline**

Digital assets need rules after first delivery.

## **Body**

Assets rarely stop after delivery.

They are reused, localized, refreshed, cut down, repackaged, translated, archived, revived, placed into paid media, added to product pages, sent to partners, or used in new campaigns.

Technology teams need a clear rule set for when assets can move again.
