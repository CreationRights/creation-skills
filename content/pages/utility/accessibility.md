## **Headline**

Accessibility matters in how media systems are built.

## **Subhead**

Creation Skills aims to provide a website experience that is clear, readable, navigable, and usable by as many visitors as possible.

## **Body**

Creation Skills works with companies around media creation processes, production agents, operating files, human approval gates, rights review, records, and Creation Rights readiness.

The website should reflect the same principle we bring to client work: important information should be clear, structured, and usable.

This Accessibility Statement explains the general accessibility approach for the Creation Skills website.

## **Note**

This page should be reviewed before publication and updated as the website is tested.

# **Section 1: Our Commitment**

## **Eyebrow**

COMMITMENT

## **Headline**

We want the website to be clear and usable.

## **Body**

Creation Skills aims to make its website accessible to visitors with different needs, devices, assistive technologies, and browsing contexts.

That includes attention to:

Readable text  
Clear page structure  
Keyboard navigation  
Descriptive links  
Helpful headings  
Sufficient contrast  
Responsive layouts  
Alternative text for meaningful images  
Forms that are clear and usable  
Content that avoids unnecessary complexity

---

# **Section 2: Accessibility Goals**

## **Eyebrow**

GOALS

## **Headline**

The site should support practical access.

## **Body**

Creation Skills intends to follow commonly accepted web accessibility practices where reasonably possible.

## **Accessibility Goals**

Pages should have clear headings.

Navigation should be understandable.

Text should be readable on desktop and mobile.

Forms should use clear labels.

Buttons and links should be identifiable.

Images should include meaningful alternative text where appropriate.

Decorative images should not create unnecessary noise for assistive technology.

Color should not be the only way information is communicated.

Interactive elements should be usable with a keyboard where possible.

The site should be reviewed as content and design evolve.

# **Section 3: Content Accessibility**

## **Eyebrow**

CONTENT

## **Headline**

Accessibility is also about language and structure.

## **Body**

Creation Skills uses structured page content so visitors can understand the offer without needing to decode vague language.

The site should make it clear:

What Creation Skills does.  
Who it helps.  
What a C/Skills Review is.  
How Creation Skills connects to Creation Rights.  
Where CreativeIP.org fits.  
What information should not be submitted through public forms.  
How visitors can contact the team.

## **Supporting Line**

Clear content is part of accessibility.

---

# **Section 4: Forms**

## **Eyebrow**

FORMS

## **Headline**

Forms should be simple to understand.

## **Body**

Creation Skills forms may include C/Skills Review requests, contact enquiries, partnership enquiries, and briefing requests.

## **Form Accessibility Goals**

Fields should have clear labels.

Required fields should be understandable.

Error messages should be useful.

Confirmation messages should be clear.

Sensitive-material warnings should be visible.

Forms should avoid asking for unnecessary information.

## **Reminder**

Visitors should not submit confidential source files, unreleased assets, contracts, private talent materials, legal documents, credentials, or sensitive rights notes through public forms.

---

# **Section 5: Images and Visual Design**

## **Eyebrow**

VISUAL DESIGN

## **Headline**

Visual design should support understanding.

## **Body**

Creation Skills may use expressive visual design, but it should not make the site harder to use.

## **Design Guidelines**

Use strong contrast between text and background.

Avoid placing important text over busy images.

Use readable type sizes.

Keep spacing generous.

Use motion carefully.

Avoid flashing or distracting visual effects.

Provide text alternatives for meaningful visuals.

Do not rely only on color to explain status, category, or action.

# **Section 6: Known Limitations**

## **Eyebrow**

LIMITATIONS

## **Headline**

Accessibility is an ongoing process.

## **Body**

The Creation Skills website may continue to evolve as pages, forms, visual systems, and tools are added.

Some accessibility issues may be discovered after launch.

Creation Skills should review and improve accessibility as the site changes.

## **Possible Areas to Review**

Navigation menus  
Form error states  
Mobile layouts  
Image alternative text  
Color contrast  
Keyboard focus states  
Embedded tools  
Third-party forms  
Analytics or scheduling widgets  
Downloadable materials if added later
