This page defines when the Creation Skills content library is ready to move from content planning into design, development, QA, and launch.

The menu structure controls what appears in navigation.

---

# **1\. Handoff Principle**

## **Headline**

Freeze the launch set before build starts.

## **Body**

The full Creation Skills library is larger than the first website build.

Before design or code begins, the team should freeze the launch set so the build does not expand during implementation.

The launch build should focus on the pages that explain the offer, support trust, route enquiries, and convert serious buyers into a C/Skills Review request.

## **Core Rule**

Master doc can stay deep.  
Launch build should stay focused.  
New pages should not be added mid-build unless they are required for legal, form, or routing function.
